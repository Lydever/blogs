一、 hash模式的实现原理: 早期的前端路由的实现是基于location.hash来实现的。其实原理很简单，location.hash的值就是URL中#后面的内容。比如下面这个网站， 它的location.hash就是'#search' 1.1 hash 路由模式的实现基于下面几个特性： 1.1.1 URL 中 hash 值只是客户端的一种状态，也就是说当向服务器端发出请求时，hash 部分不会被发送； 1.1.2 hash 值的改变，都会在浏览器的访问历史中增加一个记录。因此我们能通过浏览器的回退、前进按钮控制hash 的切换； 1.1.3 可以通过 a 标签，并设置 href 属性，当用户点击这值会发生改变；或者使用  JavaScript 来对 loaction.hash 进行赋值，改变 URL 的 hash 值； 1.1.4 我们可以使用 hashchange 事件来监听 hash 值的变化，从而对页面进行跳转（渲染）。 二、history 路与模式的实现原理： html5 提供了History API 来实现url 的变化，其中有两个最重要的api： history.pushState和history.pushResplace
